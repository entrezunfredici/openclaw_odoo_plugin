## Bonnes pratiques

- MVP volontairement minimal
- Une fonctionnalité = valeur mesurable
- Toute nouvelle fonctionnalité doit être rattachée à : un besoin + une user story + une version

## Anti-patterns à éviter

- MVP trop gros
- Versions sans objectif clair
- Ajouter “parce que facile”
- Modifier le MVP en cours sans décision tracée
